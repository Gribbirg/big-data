## Визуальное оформление
- Горизонтальная схема workflow из 5 блоков:
  ```
  [Чтение данных] → [Очистка] → [Трансформация] → [Анализ/ML] → [Результат]
  ```
- Каждый блок — прямоугольник с иконкой
- Стрелки между блоками показывают поток данных
- Под схемой: индикаторы статуса (красный/желтый/зеленый)
- Цветовое кодирование: зелёный = выполнено, желтый = в процессе, красный = ошибка

## Текст на слайде

**Визуальное программирование в KNIME**

```
Источник данных → Обработка → Анализ → Результат
```

**Workflow (рабочий процесс):**
- Узлы (Nodes) — отдельные операции
- Соединения — поток данных между узлами
- Выполнение — последовательное или параллельное

**Статусы узлов:**
- Красный — не настроен / ошибка
- Желтый — выполняется
- Зеленый — успешно завершён

## Текст выступления

Теперь разберём, как это всё работает на практике.

Основа KNIME — это workflow, то есть рабочий процесс. Он состоит из узлов и связей между ними. Каждый узел делает что-то одно: читает файл, фильтрует данные, обучает модель, строит график.

Данные как бы «текут» от одного узла к другому по соединениям — как вода по трубам. Можно строить линейные цепочки, а можно с ветвлениями и циклами — как угодно.

Очень удобная штука — индикаторы статуса под каждым узлом. Красный — что-то не так, узел не настроен или ошибка. Жёлтый — сейчас выполняется. Зелёный — всё прошло успешно.

Это сильно упрощает отладку: сразу видно, где что сломалось, и можно посмотреть данные на любом этапе.
