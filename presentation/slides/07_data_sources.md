## Визуальное оформление
- Центральная схема: KNIME в центре, вокруг — иконки источников данных
- Источники (по кругу):
  - Файлы (CSV, Excel, JSON, XML)
  - Базы данных (MySQL, PostgreSQL, Oracle)
  - Облако (AWS, Azure, Google Cloud)
  - API (REST, SOAP)
  - Big Data (Hadoop, Spark)
- Стрелки идут от источников к центру (KNIME)
- Внизу: иконки форматов экспорта

## Текст выступления

Одна из сильных сторон KNIME — работа с данными из разных источников. Реально из почти любых.

Локальные файлы — без проблем: CSV, Excel, JSON, XML, Parquet и куча других форматов.

Базы данных — MySQL, PostgreSQL, Oracle, SQL Server. NoSQL тоже поддерживается, например MongoDB.

Облачные хранилища — Amazon S3, Azure Blob Storage, Google Cloud. Это важно, потому что сейчас многие компании данные именно в облаке держат.

Для больших данных есть интеграция с Hadoop и Spark — можно ворочать терабайтами.

И ещё можно подключаться к внешним API, получать данные из веб-сервисов в реальном времени.

Всего из коробки доступно больше трёхсот коннекторов. А если чего-то не хватает — можно доставить через расширения.
